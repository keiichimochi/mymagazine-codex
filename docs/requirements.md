# 機能要件定義

## 目的

ユーザーの「脳内ゲーム史」を編集部ヒアリングとして抽出し、架空ゲーム雑誌を継続生成するための世界線エンジンを作る。

## コア要件

- 編集部ヒアリングを 10 問表示する。
- 回答内容から雑誌タイトル、世界線設定、編集方針を生成する。
- 架空ハード、架空メーカー、架空IP、年表、広告文化、ライター人格を固定DBとして扱う。
- 生成結果は「雑誌そのもの」ではなく、継続生成に使う世界線設定として出力する。
- 16ページ構成テンプレートを生成する。
- ライター人格は固定し、毎号ブレないようにする。
- ユーザーがサンプル回答を投入できる。
- 生成結果を JSON として確認できる。
- Codex App Server 経由で世界線JSONを生成できる。
- Codex App Server 経由で画像生成プロンプトを生成できる。
- Codex App Server が使えない場合はローカル生成へフォールバックする。
- 前回生成した世界線を保存し、次回以降に読み込める。
- 年月を指定して雑誌1号分を生成できる。
- 生成した雑誌を年月単位で保存・読み込みできる。
- 保存済み雑誌を一覧表示し、1クリックで開ける。
- 雑誌各ページにはソフト画面、ハード写真、広告、図解などの画像枠とキャプションを含める。
- 雑誌生成中は「Nページ目を生成中」「全体のX%」を表示する。
- 世界線の年表を表示・編集できる。
- 編集した年表が次回の雑誌生成内容に反映される。

## 非機能要件

- 静的HTMLとしても、ローカル App Server 経由でも動作する。
- Codex App Server 利用時は Codex CLI のログイン状態を利用する。
- サーバ利用時は `data/current-worldline.json` に保存する。
- 雑誌は `data/magazines/YYYY-MM.json` に保存する。
- `file://` 利用時は `localStorage` に保存する。
- 将来 AI API、画像生成、PDF出力を追加できる構成にする。
- 90年代ゲーム雑誌らしい密度、紙面感、ハード戦争感をUIに反映する。

## 対象外

- 初期版ではPDF / Kindle / 動画 / 音楽ファイルの実生成は行わない。
- 初期版では実画像ファイルの生成は行わず、画像生成プロンプトまでを作る。
- 実在企業ロゴや著作物画像は同梱しない。
