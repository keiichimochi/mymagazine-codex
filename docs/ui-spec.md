# UIデザイン仕様

## デザイン方針

90年代ゲーム雑誌の情報密度、ハード戦争感、編集部らしさを持つWebツールにする。生成される誌面は Beep メガドライブ期の攻略ページ、広告ページ、読者企画ページのような濃い紙面を参考にする。

## 画面構成

- 左側: 編集部ヒアリングフォーム
- 右側: 生成された世界線エンジン
- 下部: 雑誌、保存済み一覧、16ページ構成、DBプレビュー、年表編集、画像生成、生成パイプライン

## トーン

- 技術オタ誌
- BEEP的
- ゲーメスト的な熱量
- 攻略スクショ、色帯、赤字注釈、レビュー点数、読者投稿欄が混在する高密度紙面
- ただし操作UIは現代的に読みやすくする

## カラー

- 背景: 紙面に近い淡いグレー
- メイン: 黒、赤、青、黄色の雑誌見出し色
- アクセント: 警告帯、速報帯、レビュー点数欄を想起する色
- 雑誌ページ: 緑、黄、赤、シアンの帯をページ上部や囲み欄に使う

## コンポーネント

- テキスト入力: 複数行回答用
- ボタン: サンプル投入、世界線生成、雑誌生成、雑誌画像生成、JSONコピー
- タブ: 雑誌、保存済み、16ページ、DB、年表編集、画像生成、パイプライン、JSON
- カード: ライター、ハード、記事枠などの個別要素のみ
- 雑誌ページ: 複数の画像枠、実画像、キャプション、プロンプト、欄外メモ、点数欄を1ページ内に配置
- 画像生成枚数: Gemini API の利用量を制御するため、1回の実行で生成する素材数を指定できる

## レスポンシブ

- デスクトップでは2カラム。
- モバイルではフォーム、結果、ページ構成の順に縦積み。
- テキストがボタンやカードからはみ出さないように折り返す。
